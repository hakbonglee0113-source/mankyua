# 제주고사리삼 조사야장 — 오프라인 설치 안내

이 앱은 이제 **PWA(설치형 웹앱)** 로 만들어져서, 처음 한 번만 인터넷으로 열어두면
그 다음부터는 **인터넷이 전혀 안 되는 산속·오지 현장에서도** 완전히 작동해요.
입력한 데이터도 인터넷 없이 휴대폰 안(IndexedDB)에 그대로 저장됩니다.

## 포함된 파일 (5개, 반드시 같은 폴더에 함께 있어야 함)
- `index.html` — 앱 본체
- `manifest.json` — 설치 정보(앱 이름, 아이콘 등)
- `sw.js` — 오프라인 캐싱 담당 서비스워커
- `icon-192.png`, `icon-512.png` — 앱 아이콘

## 1. 배포 (GitHub Pages 추천 — 가장 안정적)
1. 기존에 쓰시던 GitHub Pages 저장소(또는 새 저장소)에 위 5개 파일을 **저장소 루트에 그대로** 업로드
2. `https://아이디.github.io/저장소이름/` 주소로 접속 확인 (index.html이 자동으로 열림)

## 2. 폰에서 설치 (Wi-Fi나 데이터가 될 때 딱 한 번만 하면 됨)
**Android (Chrome)**
1. 위 주소로 접속
2. 하단에 "홈 화면에 추가" 또는 "앱 설치" 배너가 뜨면 탭 → 설치
   (안 뜨면 오른쪽 위 메뉴(⋮) → "앱 설치"/"홈 화면에 추가")
3. 홈 화면에 아이콘 생김 → 이제부터는 그 아이콘으로 실행

**iPhone (Safari)**
1. 위 주소로 접속
2. 공유 버튼(□↑) → "홈 화면에 추가"
3. 홈 화면에 아이콘 생김

## 3. 오프라인 확인
- 설치 후 **비행기 모드**를 켠 상태로 홈 화면 아이콘을 눌러 앱이 정상적으로 열리는지 한 번 확인해보세요.
- 화면 우측 상단에 "오프라인" 표시가 뜨는 건 정상이에요 (인터넷 연결 상태만 알려주는 표시일 뿐, 앱 사용에는 전혀 지장 없어요).
- 조사 기록 입력·저장·불러오기·음성입력·텍스트 내보내기 모두 인터넷 없이 그대로 작동합니다.

## 3-1. 엑셀 / PDF 내보내기 (신규, 오프라인에서도 작동)
"메모·내보내기" 탭에 버튼이 추가됐어요. 둘 다 외부 프로그램이나 인터넷 없이 휴대폰 자체 기능만으로 만들어져서 오프라인에서도 그대로 작동해요.

- **📊 엑셀로 내보내기 (.xls)** — 기본정보 / 방형구별 요약 / 식물상 상세(무한추가분 전부 한 줄씩) / 암석 상세(무한추가분 전부 한 줄씩), 총 4개 시트로 저장돼요. PC나 폰 엑셀 앱에서 열 때 "파일 형식이 확장자와 다릅니다" 경고가 뜰 수 있는데, **"예/열기"**를 누르면 정상적으로 열립니다.
- **🖨️ PDF로 내보내기** — 누르면 인쇄 미리보기 화면이 열려요. 프린터 선택 목록에서 **"PDF로 저장"**을 고르면 휴대폰에 PDF 파일로 저장돼요. (Android Chrome 기준. 팝업 차단 알림이 뜨면 허용 후 다시 눌러주세요.)

## 4. 주의할 점
- ⚠️ **처음 설치할 때는 반드시 인터넷이 한 번 연결돼 있어야** 앱 내용을 캐시에 받아둘 수 있어요. (그 다음부터는 인터넷 필요 없음)
- ⚠️ 데이터는 이 폰의 이 브라우저에만 저장돼요 (기기 간 자동 동기화 안 됨) — 조사 회차별로 **"📄 텍스트로 내보내기" / "💾 JSON 저장"** 버튼을 눌러 구글드라이브 등에 백업해두는 걸 추천해요.
- ⚠️ 나중에 앱 내용을 수정해서 다시 올릴 때는 `sw.js` 안의 `CACHE_NAME` 값을 반드시 올려주세요 (이번 엑셀·PDF 기능 추가로 이미 `v1` → `v2`로 올려뒀어요). 그래야 사용자가 인터넷이 연결됐을 때 새 버전으로 자동 업데이트돼요. 이 부분은 앞으로 수정해드릴 때마다 제가 같이 챙겨드릴게요.
  → 즉, **기존에 이미 설치하신 상태라면 Wi-Fi가 될 때 한 번 앱을 다시 열어주셔야** 이번에 추가된 엑셀·PDF 버튼이 반영돼요.

## (선택) 인터넷 없이 그냥 파일만 열어서 쓰기
GitHub Pages 없이 `index.html` 파일 자체를 폰에 저장해서 파일탐색기로 직접 열어도 작동은 해요.
다만 이 경우 서비스워커/설치가 정상 동작하지 않을 수 있고(파일 경로가 바뀌면 저장된 데이터가 끊길 위험),
Chrome의 파일(`file://`) 접근 정책에 따라 일부 기능(음성입력 등)이 제한될 수 있어
**GitHub Pages로 배포하는 방법을 권장**드려요.
